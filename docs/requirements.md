# 需求分析文档

## 1. 项目概述

本项目目标是搭建一个个人静态博客，部署在 GitHub Pages 上，站点地址为 `https://LingengMa.github.io/`。博客风格参考 [rqdmap.top](https://rqdmap.top/)，以技术写作为主，兼顾生活记录。

---

## 2. 目标用户

- **主要读者**：对技术（编程、Linux、系统安全等）感兴趣的中文读者
- **作者自身**：知识沉淀与个人记录

---

## 3. 功能需求

### 3.1 内容分区

| 分区 | 路径 | 说明 |
|------|------|------|
| 首页 | `/` | 个人介绍、社交链接、近期动态（Git 活跃热力图） |
| 博客 | `/posts/` | 通用技术文章，支持标签/分类 |
| 关于 | `/about/` | 个人介绍页面 |
| 日志 | `/changelog/` | 站点更新日志，记录博客内容的变更历史 |

### 3.2 文章功能

- **Markdown 写作**：支持标准 Markdown + Hugo shortcode
- **代码高亮**：语法高亮，代码块带复制按钮
- **数学公式**：KaTeX 渲染（行内 `$...$`，块级 `$$...$$`）
- **目录（TOC）**：自动生成文章目录，随滚动高亮当前章节
- **标签与分类**：文章支持 `tags` 和 `categories` 元数据
- **草稿模式**：`draft: true` 的文章仅本地预览时可见

### 3.3 站点功能

- **全文搜索**：客户端全文搜索（由 inkwell 主题内置提供）
- **排序切换**：文章列表支持按创建时间、修改时间、字数排序
- **RSS 订阅**：自动生成 `/index.xml`
- **深色/浅色模式**：由主题提供（若 inkwell 支持）

### 3.4 首页特性（参考 rqdmap.top）

- **热力图**：展示按年度的 Git 提交活跃度，可切换年份
- **社交链接**：邮件、GitHub 等图标链接

---

## 4. 非功能需求

### 4.1 性能

- 纯静态页面，无需服务端，访问延迟低
- 图片由作者手动优化，主题不引入重型 JS 框架

### 4.2 SEO

- 每篇文章有独立 `<title>` 和 `<meta>` 描述
- 自动生成 `sitemap.xml`

### 4.3 可维护性

- 内容与样式分离：文章以 Markdown 管理，样式由主题负责
- 新增文章只需在 `content/posts/` 下新建 `.md` 文件
- 构建一键完成（`hugo` 命令）

### 4.4 国际化

- 主语言：中文简体（`zh-cn`）
- 日期、界面文案均使用中文

---

## 5. 约束条件

- **零服务端成本**：仅依赖 GitHub Pages（免费静态托管）
- **主题不修改**：inkwell 以 git submodule 引入，样式定制通过 Hugo 配置或覆盖机制实现
- **无数据库**：所有内容以 Markdown 文件形式存储在 Git 仓库中
